async await基本用法

发表于 2019-12-22 更新于 2019-12-30 分类于 JavaScript ， 学习笔记 阅读次数：
async/await从字面意思上很好理解，async是异步的意思，await有等待的意思，而两者的用法上也是如此。async用于申明一个function是异步的，而await 用于等待一个异步方法执行完成。

  • 另外还有一个很有意思的语法规定，await 只能出现在 async 函数中，否则会报语法错误。

async 的作用

async 函数（包含函数语句、函数表达式、Lambda表达式）会返回一个 Promise 对象，如果在函数中 return 一个表达式，async 会把这个表达式通过 Promise.resolve() 封装成 Promise 对象,可以使用then方法添加回调函数。

如果 async 函数没有返回值，它会返回 Promise.resolve(undefined)。

感觉和普通函数没什么区别，只是包裹里一层 promise 和 关键字 async 。看来所有的奥秘都在 awiat 。

await

  • await是 async wait 的意思，wait的是异步数据的返回值，并把数据data返回。相当于阻塞了代码，使代码顺序执行，表现和同步代码几乎一模一样。

    可以看出，fetch方法 比 test函数里的fetch少嵌套了一层 then，但却拿到了数据。个人感觉 await 拿到了 Promise 的 [[PromiseValue]], 这个就是我们需要的数据，但现在它是 Promise的内部数据，需要 .then 才能取到。所以我觉得，await 其实把返回的 Promise 对象多调用了一层 then，拿到了数据。

  • 当函数执行的时候，一旦遇到 await 表达式，等待 await 后面的表达式运行完并且有了返回结果之后，才继续执行函数体内后面的语句，以达到同步的效果。await 在写法上也是同步的。

  • await 后面的表达式可以是普通函数调用或者变量，跟着什么是什么，没有特殊限定。

有了这个async关键字，只是表明里面可能有异步过程，里面可以有await关键字。当然，全部是同步代码也没关系。当然，这时候这个async关键字就显得多余了。不是不能加，而是不应该加。

异常处理

await 只关心异步过程成功的消息 resolve(data)，拿到相应的数据data。至于失败消息reject(error)，不关心，不处理。
当然对于错误消息的处理，有以下几种方法供选择：

  • 像同步代码一样，放在一个try…catch结构中

  • 让 await 后面的 Promise对象自己catch

  • 也可以让外面的async函数返回的Promise对象统一catch

总结

  • async函数的返回值是一个Promise对象，这个是和普通函数本质不同的地方。这也是使用时重点注意的地方。

  • async函数内部只要有一个异步过程发生错误，整个执行过程就中断，这个返回的Promise对象的catch就能抓到这个错误。

  • async函数执行和普通函数一样，函数名带个()就可以了，参数个数随意，没有限制；也需要有async关键字。只是返回值是一个Promise对象，可以用then函数得到返回值，用catch抓去整个流程中发生的错误。
