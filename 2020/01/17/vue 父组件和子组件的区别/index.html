vue 父组件和子组件的区别

发表于 2020-01-17 分类于 vue ， 学习笔记 阅读次数：
重新认识 vue 的父子组件。

父组件和子组件基本区别

  • 看了很多篇文章，基本都是说，#app 是根组件，也是所有组件的父组件。包裹在里面的组件是其 子组件。

  • 也即是说看组件的父元素节点。

  • 如果 home 组件模板中也包裹着 <children></children>, 也就是说 home 是 父组件，children 是子组件。

个人理解的父子组件: 看数据流向

  • 在官网文档中用 props 向 子组件传值。

    以前我也是一度 以为使用的组件的地方是父组件，定义的地方是子组件，虽然他们是同一个组件。

  • 其实 组件名字 在我看来是一个桥梁，沟通父子组件之间的数据。看数据的流向，就可以知道谁是父组件，谁是子组件。
    向上述代码 中 <home :use="msg"></home> , msg 来自 根组件 的 data ，home 中 用 props 来接受数据。这样看 根组件是父组件，homn是 子组件。

    再如:<children :get='child'></children> 中的 child 来自 home 的 data， children 中 用 props 来接受数据。也是可以一目了然的。
