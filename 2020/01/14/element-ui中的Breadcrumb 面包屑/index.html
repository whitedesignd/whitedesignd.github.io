vue 之 element 的动态 Breadcrumb(面包屑)

发表于 2020-01-14 分类于 vue ， element ui 阅读次数：
在element中Breadcrumb(面包屑)，希望能复用，抽离代码封装成一个组件，做成动态面包屑。而在 element 中的 el-menu 中 用 自带的 router 属性，用 el-menu-item 的 index 作为 path 进行路由跳转( 或者使用语义化更好的 route 属性 来进行路由跳转，不过设置路由高亮比较麻烦），不能携带数据跳转。又不想用 routeLink 来实现。所以 采用了 路由元信息 来进行 数据信息的传递。

vue 之 element 的动态 Breadcrumb(面包屑)

面包屑,一般情况下它用来表示我们当前所处的站点位置，也可以帮助我们能够更快的回到上个层级。用来展示系统的结构（即我在哪），而不是访问历史。

最主要的还是组件复用。取出 路由 的 meta 字段信息，来做面包屑，而不是写死内容，多些了很多结构。

  • 笨笨的方式，就 element 官网的例子，数据固定，或者 在 data 中写数据，用 v-for循环出来。

这样写就 需要在每个页面 都 这样的结构添加，非常麻烦带来的维护成本是巨大的。所以 利用 路由元信息。

使用路由元信息 meta

用 $route.matched 来获取 meta 。

路由对象 matched 属性: 包含当前路由的匹配到的嵌套路径片段的路由记录。

matched：返回一个数组，包含当前路由的所有嵌套路径片段的路由记录。路由记录就是 routes 配置数组中的对象副本 (如果匹配到子路由，则还会包含 children 数组)。

以下就是 this.$route.matched[0] 获取到的 路由记录，包含了我们需要的 meta 字段。” > “ 代表可以展开，信息很多，请自行了解。

原因：路由路径重复报错，vue-router3.1 后 push 和 replace 方法返回 promise。promise 被 reject 但是你没有catch，所以报错 Uncaught。3.1版本之前是通过传回调函数捕获错误，没有回调就不会捕获也不会报错。

三种解决办法:

  • 在项目目录下运行 npm i vue-router@3.0 -S 即可。

  • 由于 push 和 replace 方法返回 promise，所以 捕捉错误即可
